## NOTES

The root cause of the cross-session bug: `CreateVersion()` called `ListFilesByPath()` which returned files from **all sessions** for a given path. It took `max(version)` globally and added 1, but the UNIQUE constraint is `(path, session_id, version)` — scoped per-session. This meant version numbers from session A leaked into session B.

### Fix

Added a new SQL query `ListFilesByPathAndSession` that filters by both `path` AND `session_id`. Changed `CreateVersion()` to use it instead of `ListFilesByPath`.
