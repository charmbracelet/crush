# ADR-000: Architecture Decision Records

## Status

Accepted

## Context

We need a way to document significant architectural decisions for the Orchestra system.

## Decision

We will use Architecture Decision Records (ADRs) to document key decisions.

ADR format:
- Title: Short noun phrase
- Status: Proposed, Accepted, Deprecated, Superseded
- Context: Why this decision is needed
- Decision: What we decided
- Consequences: Trade-offs and impact

## Consequences

- Decisions are documented and traceable
- Future contributors understand the "why"
- Easier to revisit decisions when context changes
