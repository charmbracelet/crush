## 一、系統總覽

### 1.1 系統定位

CrushCL 多代理協作系統是一個基於 Go 語言的輕量級多代理框架，專為 Windows 環境優化。系統參考 Claude Code 的架構啟發，實現真正的代理間協同工作能力。

### 1.2 核心目標

| 目標 | 說明 |
|------|------|
| **訊息傳遞** | 構建可靠的代理間訊息總線 |
| **狀態管理** | 實現事件驅動的狀態機 |
| **故障保護** | 提供心跳檢測和熔斷機制 |
| **真正協作** | 讓代理可以互相配合並真實的協同工作 |

### 1.3 約束條件

- **Windows 優先**: 所有組件必須支援 Windows
- **語言**: 繁體中文文檔
- **Go 模組**: `github.com/charmbracelet/crushcl`

```
    │
```

```
    └─── MessageBus (狀態變化發布)

SwarmExt
    │
    ├─── Coordinator (協調)
    │
    ├─── MessageBus (通信)
    │
    ├─── StateMachine (狀態管理)
    │
    └─── Guardian (故障處理)

    │

CheckpointSystem
    │
    └─── MessageBus (故障恢復)
```

```
├── consensus/           # 共識協議 [待建]
│   └── protocol.go
├── checkpoint/          # 檢查點系統 [待建]
│   └── system.go
└── ui/                 # 用戶介面 [待建]

```

### 7.2 檔案命名

| 類型 | 規範 | 範例 |
|------|------|------|
| Go 源碼 | `snake_case.go` | `state_machine.go` |
| 測試檔案 | `*_test.go` | `messagebus_test.go` |
| 文檔 | `*.md` | `README.md` |

### 7.3 類型命名

| 類型 | 規範 | 範例 |
|------|------|------|
| 結構體 | `PascalCase` | `MessageBus`, `StateMachine` |
| 介面 | `PascalCase` | `Publisher`, `Subscriber` |
| 方法 | `PascalCase` | `Send()`, `Broadcast()` |
| 欄位 | `camelCase` | `isOpen`, `agentId` |
| 常數 | `PascalCase` 或 `ALL_CAPS` | `TypeTaskFailed`, `MAX_RETRIES` |

## 九、構建與部署

### 9.1 編譯命令

```bash
# 全部編譯
go build ./...

# 運行測試
go test ./internal/agent/... -v

# 生成測試覆蓋報告
go test ./internal/agent/... -coverprofile=coverage.out
go tool cover -html=coverage.out -o coverage.html
```

### 9.2 交叉編譯 (Windows)

```bash
# Windows AMD64
GOOS=windows GOARCH=amd64 go build -o crushcl.exe ./cmd/crush

# Windows ARM64
GOOS=windows GOARCH=arm64 go build -o crushcl.exe ./cmd/crush
```
